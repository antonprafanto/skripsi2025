## BAGIAN 3: AUDIT METODOLOGI PENELITIAN

### ‚úÖ KEKUATAN METODOLOGI

1. **IMSDD sebagai Framework Solid**
   - Pemilihan metode sesuai dengan nature pengembangan AR multimedia
   - 4 tahapan jelas dan terstruktur
   - Iterative process memungkinkan improvement

2. **Pengujian Dua Tahap**
   - Beta testing untuk validasi teknis
   - UAT untuk validasi user acceptance
   - Kombinasi ini komprehensif

3. **Ukuran Sampel Cukup**
   - 44 responden memadai untuk penelitian skripsi
   - Diversitas responden (usia, pendidikan)

### ‚ö†Ô∏è KELEMAHAN METODOLOGI

1. **Validitas Instrumen** (PRIORITAS TINGGI)

   **Masalah**:
   - Kuesioner UAT tidak melalui validasi ahli atau pilot test
   - Tidak ada uji reliabilitas (Cronbach's Alpha)
   - Pertanyaan kuesioner tidak dilampirkan lengkap

   **Dampak**:
   - Hasil UAT mungkin tidak reliabel
   - Pertanyaan mungkin bias atau ambigu

   **Rekomendasi**:
   - Lakukan expert validation dengan minimal 2 ahli
   - Pilot test dengan 5-10 responden
   - Hitung Cronbach's Alpha (target >0.7)
   - Lampirkan instrumen lengkap

2. **Sampling Bias** (PRIORITAS SEDANG)

   **Masalah**:
   - Purposive sampling rentan bias
   - Kriteria responden terlalu luas
   - Tidak ada kontrol untuk variabel confounding

   **Dampak**:
   - Hasil tidak generalizable untuk seluruh populasi Samarinda

   **Rekomendasi**:
   - Gunakan stratified sampling untuk representasi lebih baik
   - Atau tetap purposive tapi acknowledge limitasi di pembahasan
   - Control variabel: pendidikan, pengalaman teknologi

3. **Tidak Ada Control Group** (PRIORITAS RENDAH)

   **Masalah**:
   - Tidak ada pembanding dengan media pembelajaran tradisional
   - Tidak bisa mengukur efektivitas relatif AR vs metode lain

   **Dampak**:
   - Tidak bisa claim AR lebih efektif dari media lain

   **Rekomendasi**:
   - Acknowledge sebagai limitasi
   - Atau tambahkan quasi-experiment dengan control group (jika waktu memungkinkan)

4. **Metrik Evaluasi Terbatas** (PRIORITAS SEDANG)

   **Masalah**:
   - Hanya mengukur acceptance, tidak measuring learning outcome
   - Tidak ada pre-test post-test untuk knowledge gain
   - Tidak mengukur retention (apakah knowledge bertahan?)

   **Dampak**:
   - Tidak bisa klaim aplikasi efektif meningkatkan pengetahuan

   **Rekomendasi**:
   - Tambahkan pre-test post-test design (jika feasible)
   - Atau fokus claim pada "acceptance" bukan "effectiveness"
   - Revisi tujuan penelitian agar aligned dengan metrik
