**Contoh Pseudocode untuk Animasi Siklus:**

```
ALGORITHM: PlayCyclicAnimation()

INPUT: animationSpeed (float), loopMode (boolean)
OUTPUT: Animated 4-stroke cycle

BEGIN
  currentPhase = INTAKE
  timer = 0

  WHILE (userPressedStop == false) DO
    CASE currentPhase OF
      INTAKE:
        MovePiston(DOWN, animationSpeed)
        OpenValve(INTAKE_VALVE)
        CloseValve(EXHAUST_VALVE)
        DisplayLabel("LANGKAH ISAP")
        IF (pistonReachedBottom) THEN
          currentPhase = COMPRESSION
        END IF

      COMPRESSION:
        MovePiston(UP, animationSpeed)
        CloseValve(INTAKE_VALVE)
        CloseValve(EXHAUST_VALVE)
        DisplayLabel("LANGKAH KOMPRESI")
        IF (pistonReachedTop) THEN
          currentPhase = POWER
        END IF

      POWER:
        TriggerSparkPlug()
        MovePiston(DOWN, animationSpeed * 1.5)  // Faster due to explosion
        CloseValve(INTAKE_VALVE)
        CloseValve(EXHAUST_VALVE)
        DisplayLabel("LANGKAH USAHA")
        IF (pistonReachedBottom) THEN
          currentPhase = EXHAUST
        END IF

      EXHAUST:
        MovePiston(UP, animationSpeed)
        CloseValve(INTAKE_VALVE)
        OpenValve(EXHAUST_VALVE)
        DisplayLabel("LANGKAH BUANG")
        IF (pistonReachedTop) THEN
          IF (loopMode == true) THEN
            currentPhase = INTAKE  // Restart cycle
          ELSE
            EXIT  // Stop after 1 cycle
          END IF
        END IF
    END CASE

    timer = timer + deltaTime
  END WHILE
END
```

**2. Tips Menjawab Pertanyaan:**

**Q: "Apa bedanya penelitian Anda dengan penelitian [X] yang juga AR markerless untuk mesin motor?"**
A: "Penelitian [X] fokus pada visualisasi komponen statis. Penelitian saya fokus pada visualisasi SIKLUS DINAMIS 4 langkah dengan animasi real-time, plus dirancang khusus untuk kurikulum TKRO SMK Indonesia dengan testing usability komprehensif. Selain itu, saya menambahkan fitur [fitur unik Y]."

**Q: "Bagaimana Anda memastikan animasi 3D akurat secara teknis?"**
A: "Saya melakukan observasi langsung di bengkel SMK dan wawancara dengan guru TKRO untuk memahami detil mekanik. Animasi akan di-review oleh guru TKRO sebelum testing dengan siswa. Saya juga merujuk pada manual teknis mesin motor [merek X] sebagai referensi."

**Q: "Kenapa tidak pakai Unity ML atau AI untuk tracking?"**
A: "Vuforia sudah memiliki algoritma NFT (Natural Feature Tracking) yang mature dan terbukti stabil untuk markerless AR. Menggunakan ML/AI akan menambah kompleksitas dan resource consumption yang berat untuk device Android entry-level yang menjadi target. Fokus penelitian ini adalah pada visualisasi edukatif, bukan inovasi algoritma tracking."

**Q: "Bagaimana jika tracking tidak stabil di kondisi pencahayaan buruk?"**
A: "Saya sudah mengantisipasi ini di Risk Management (BAB III halaman XX). Strategi mitigasi: (1) Menyediakan 2-3 target image alternatif dengan rating Vuforia tinggi, (2) Testing di berbagai kondisi pencahayaan sejak tahap awal, (3) Menambahkan panduan di aplikasi untuk kondisi pencahayaan optimal."

**Q: "Timeline 6-7 bulan realistis tidak?"**
A: "Berdasarkan Gantt chart di BAB III, saya sudah mengalokasikan waktu per tahapan secara detail. Fase paling padat adalah Bulan 3-4 (3D modeling + assembly). Saya juga sudah menyediakan buffer 2 minggu untuk revisi. Jika ada kendala teknis, saya sudah menyiapkan contingency plan dengan prioritas fitur Must-Have vs Nice-to-Have (BAB III halaman XX)."
