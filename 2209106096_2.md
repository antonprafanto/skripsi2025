### 5.2 Kelemahan Metodologis Prinsipil

> **[KRITIS]** **Sequential Search vs. MySQL WHERE tidak seimbang secara teoritis.** MySQL WHERE menggunakan B-Tree index dengan kompleksitas O(log n), sedangkan Sequential Search O(n). Perbandingan ini tidak adil kecuali kolom pencarian tidak diindeks. Harus dijelaskan secara eksplisit apakah tabel database menggunakan index atau tidak saat pengujian.

> **[KRITIS]** **Merge Sort PHP vs. MySQL ORDER BY tidak seimbang.** MySQL ORDER BY menggunakan algoritma sort yang sangat dioptimasi di level C (kombinasi introsort). PHP-level Merge Sort pasti lebih lambat. Justifikasi mengapa perbandingan ini relevan harus diperkuat secara signifikan di latar belakang.

> **[SEDANG]** **Tidak ada analisis kompleksitas ruang (*space complexity*).** Merge Sort membutuhkan O(n) ruang tambahan, penting dalam konteks server web dengan memori terbatas. Ini harus dibahas.

> **[SEDANG]** **Scope penelitian perlu diklarifikasi.** Saat ini terkesan menggabungkan dua tujuan: membangun *sistem* yang fungsional SEKALIGUS *mengevaluasi algoritma*. Pastikan keduanya dapat dicapai dalam satu skripsi atau salah satunya menjadi fokus utama.
