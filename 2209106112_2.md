## 1. RINGKASAN TOPIK YANG DIAJUKAN

Secara sederhana, Adam ingin membuat **sistem penyiram tanaman otomatis** yang:

- Dapat membaca kelembapan tanah menggunakan sensor kapasitif
- Melakukan penyiraman hanya pada waktu yang optimal (pagi/sore hari)
- Dapat dipantau secara jarak jauh melalui aplikasi atau website
- Ditargetkan untuk tanaman hortikultura skala pot/polybag

**Teknologi yang digunakan:**

| Komponen | Fungsi |
|----------|--------|
| ESP32 | Mikrokontroler utama dengan kemampuan WiFi |
| Sensor Kelembapan Kapasitif | Mengukur kadar air dalam tanah |
| Relay + Pompa Mini DC | Aktuator untuk mengalirkan air |
| NTP/RTC | Sinkronisasi waktu untuk time window |
| Aplikasi/Web | Antarmuka monitoring jarak jauh |

**Kebaruan yang ditawarkan:**
- Algoritma kontrol hibrida (kelembapan tanah + time window)
- Mekanisme hysteresis untuk stabilisasi keputusan
- Pengujian terukur pada skala pot/polybag

## 4. ASPEK YANG PERLU DIPERBAIKI

### a) Kebaruan (Novelty) Perlu Diperkuat

**Permasalahan:**
Sistem irigasi otomatis berbasis IoT merupakan topik yang sudah banyak diteliti. Kebaruan yang ditawarkan (kontrol hibrida + hysteresis) secara algoritmik tidak terlalu kompleks.

Logika dasarnya relatif sederhana:
```
JIKA (kelembapan < threshold_bawah) DAN (waktu dalam range pagi/sore)
    MAKA aktifkan pompa
JIKA (kelembapan > threshold_atas)
    MAKA matikan pompa
```

**Saran Perbaikan:**
Perkuat kontribusi di sisi software dengan menambahkan:
- Dashboard monitoring dengan visualisasi grafik history kelembapan
- Sistem logging data ke database untuk analisis pola
- Fitur notifikasi (push notification/email) untuk kondisi anomali
- Laporan statistik penggunaan air dan efektivitas penyiraman

### b) Porsi Software vs Hardware Perlu Diseimbangkan

**Permasalahan:**
Dari deskripsi saat ini, skripsi terkesan lebih dominan pada aspek hardware (merangkai komponen) daripada aspek software (pemrograman).

**Mengapa ini penting?**
Program Studi Informatika seharusnya menekankan kompetensi software development. Aspek hardware sebaiknya menjadi pendukung, bukan fokus utama.

**Saran Perbaikan:**
- Kembangkan aplikasi monitoring secara mandiri (bukan hanya menggunakan platform seperti Blynk)
- Implementasikan backend dengan database untuk menyimpan data historis
- Buat dokumentasi algoritma yang komprehensif (flowchart, pseudocode)
- Tambahkan fitur analisis data dari hasil monitoring

### c) Rumusan Masalah Terdapat Overlap

**Permasalahan:**
Rumusan masalah #1 dan #2 memiliki kemiripan:

- **RM #1:** "Bagaimana merancang bangun sistem irigasi pintar... dengan memanfaatkan data sensor kelembapan tanah?"
- **RM #2:** "Bagaimana merancang dan mengimplementasikan algoritma kontrol hibrida yang mengintegrasikan data sensor kelembapan tanah dan penjadwalan waktu?"

Keduanya sama-sama membahas perancangan sistem dengan sensor kelembapan.

**Saran Perbaikan:**
Pertimbangkan untuk merevisi rumusan masalah agar lebih distingtif:
- RM #1: Fokus pada arsitektur sistem secara keseluruhan
- RM #2: Fokus spesifik pada algoritma kontrol hibrida
- RM #3: Fokus pada evaluasi dan pengujian (sudah baik)

### d) Metrik Pengujian Belum Spesifik

**Permasalahan:**
Rumusan masalah #3 menyebutkan evaluasi berdasarkan:
- Stabilitas pembacaan sensor
- Ketepatan penyiraman sesuai time window
- Keberhasilan monitoring real-time

Namun tidak ada indikator terukur yang spesifik.

**Saran Perbaikan:**
Tambahkan metrik kuantitatif, contoh:

| Parameter | Metrik yang Disarankan |
|-----------|----------------------|
| Stabilitas sensor | Standar deviasi pembacaan < 5% pada kondisi statis |
| Ketepatan waktu | Delay aktivasi pompa < 5 detik dari threshold |
| Responsivitas monitoring | Data ter-update dalam interval < 10 detik |
| Akurasi kelembapan | Korelasi dengan pengukuran manual > 90% |

### e) Kontribusi Penelitian Perlu Diperjelas

**Permasalahan:**
Bagian 1.6 (Kontribusi Penelitian) menyebutkan kontribusi berupa "rancangan dan implementasi algoritma kontrol hibrida", namun belum menjelaskan secara konkret apa yang membedakan dengan penelitian sejenis.

**Saran Perbaikan:**
Perjelas kontribusi dengan menyebutkan:
- Apa output konkret yang dihasilkan? (prototipe, source code, dataset, dll.)
- Bagaimana algoritma hibrida ini berbeda dari pendekatan sebelumnya?
- Apakah ada parameter/threshold baru yang ditemukan?
