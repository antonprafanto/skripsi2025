- CART cocok untuk **real-time decision making** karena kompleksitas komputasi yang rendah

def explain_prediction(model, input_data):
    # Get decision path
    node_indicator = model.decision_path(input_data)
    leaf_id = model.apply(input_data)

    # Extract feature and threshold at each node
    feature = model.tree_.feature
    threshold = model.tree_.threshold

    # Build explanation
    explanation = []
    for node_id in node_indicator:
        if feature[node_id] != _tree.TREE_UNDEFINED:
            feature_name = feature_names[feature[node_id]]
            threshold_value = threshold[node_id]

            if input_data[feature[node_id]] <= threshold_value:
                explanation.append(f"{feature_name} <= {threshold_value}")
            else:
                explanation.append(f"{feature_name} > {threshold_value}")

    return explanation

```
2.3.4 Analisis Kompleksitas Algoritma CART

A. Time Complexity

1. Training Phase:
   - Kompleksitas: O(n √ó m √ó log n)
   Di mana:
   - n = jumlah sampel data training
   - m = jumlah fitur
   - log n = tinggi pohon (worst case)

   Penjelasan:
   - Untuk setiap node, algoritma menghitung Gini Index untuk semua fitur (O(m))
   - Untuk setiap fitur, perlu sort data untuk menemukan split terbaik (O(n log n))
   - Proses rekursif hingga leaf nodes tercapai (depth ‚âà log n pada balanced tree)

2. Prediction Phase:
   - Kompleksitas: O(h)
   Di mana h = tinggi pohon (height of tree)

   Penjelasan:
   - Hanya perlu traverse dari root ke leaf
   - Pada setiap node, hanya evaluasi 1 kondisi (O(1))
   - Worst case: h = log n (balanced tree)
   - Best case untuk real-time: h bisa dibatasi dengan max_depth

   Untuk kasus ini:
   - Jika max_depth = 5, maka prediction time = O(5) = O(1) ‚Üí SANGAT CEPAT!

B. Space Complexity

1. Model Size:
   - Kompleksitas: O(k)
   Di mana k = jumlah node dalam pohon

   Untuk kasus ini:
   - Fitur: 4 (pH, soil_moisture, temperature, humidity)
   - Max depth: 5
   - Worst case nodes: 2^5 - 1 = 31 nodes
   - Setiap node menyimpan: feature_id, threshold, gini, samples
   - Estimasi size: 31 nodes √ó 32 bytes = ~1 KB ‚Üí SANGAT KECIL!

2. Memory Usage saat Prediction:
   - O(h) untuk stack recursion
   - Untuk max_depth=5: hanya butuh 5 √ó beberapa bytes ‚Üí SANGAT EFISIEN!

C. Perbandingan Kompleksitas dengan Algoritma Lain

| Algoritma | Training Time | Prediction Time | Model Size |
|-----------|---------------|-----------------|------------|
| CART | O(n√óm√ólog n) | O(log n) | O(k) = ~1 KB |
| C4.5 | O(n√óm√ólog n) | O(log n) | O(k) = ~1 KB |
| Random Forest | O(t√ón√óm√ólog n) | O(t√ólog n) | O(t√ók) = ~100 KB |
| Naive Bayes | O(n√óm) | O(m) | O(m√óc) = ~0.1 KB |
| KNN | O(1) | O(n√óm) | O(n√óm) = besar |

Di mana: t = jumlah trees, c = jumlah class

Kesimpulan:
- CART memiliki prediction time O(log n) ‚Üí CEPAT untuk real-time
- Model size sangat kecil ‚Üí COCOK untuk IoT dengan limited storage
- Trade-off: training time O(n√óm√ólog n) ‚Üí tidak masalah karena training dilakukan sekali

D. Implikasi untuk Sistem IoT

Mengapa CART cocok untuk IoT real-time monitoring:

1. Fast Prediction: O(log n) ‚âà O(1) dengan depth terbatas
   ‚Üí Latency < 1ms untuk inference
   ‚Üí Cocok untuk real-time requirement

2. Small Model Size: ~1 KB
   ‚Üí Bisa di-load di ESP32 (jika perlu edge computing)
   ‚Üí Bandwidth minimal untuk download model update

3. Low Memory: O(log n) memory saat inference
   ‚Üí Tidak perlu RAM besar
   ‚Üí Bisa jalan di mikrokontroler

4. Interpretable: Tidak memerlukan komputasi kompleks
   ‚Üí User bisa memahami keputusan tanpa overhead komputasi
```
