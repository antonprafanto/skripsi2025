## Yang Harus Dibuat — Penjelasan untuk Mahasiswa

### 1. Abstrak (maks. 250 kata)

Abstrak ditulis **terakhir** (setelah Bab IV selesai), tapi untuk proposal bisa buat **draft sementara**.

**Struktur abstrak yang benar:**
1. **Masalah** — 1–2 kalimat (pelayanan manual di kelurahan)
2. **Tujuan** — 1 kalimat (membangun sistem berbasis WhatsApp)
3. **Metode** — 1–2 kalimat (Waterfall, observasi, Black Box, UAT)
4. **Hasil** — 1–2 kalimat (isi setelah sistem selesai; untuk proposal: "sistem dirancang dengan…")
5. **Kesimpulan** — 1 kalimat

**Contoh draft untuk proposal:**
> Pelayanan surat administrasi di Kelurahan Kampung Sambakungan masih dilakukan secara manual sehingga masyarakat harus datang langsung dan mengantre. Penelitian ini bertujuan merancang dan membangun sistem layanan surat administrasi berbasis WhatsApp terintegrasi panel admin web. Metode pengembangan yang digunakan adalah Waterfall dengan teknik pengumpulan data observasi, wawancara, dan kuesioner. Pengujian dilakukan menggunakan Black Box Testing dan User Acceptance Testing (UAT). Sistem dirancang menggunakan Laravel, MySQL, dan WhatsApp Cloud API dengan fitur pengajuan surat, pelacakan status, dan notifikasi otomatis.

**Kata kunci:** Sistem Informasi, WhatsApp, Pelayanan Publik, Kelurahan, Waterfall
