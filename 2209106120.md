### 4.5 Perancangan Proses/Algoritma: **60/100 (C)**

**Algoritma yang Disebutkan:**
1. Inisialisasi perangkat dan Wi-Fi
2. Baca data dari MAX30102
3. Hitung BPM dengan rumus: BPM = 60/T
4. Tampilkan di OLED
5. Kirim ke IoT platform

**ANALISIS KRITIS:**
‚ùå **Terlalu high-level** - tidak ada detail implementasi
‚ùå **Tidak ada algoritma filtering** - mentah dari sensor langsung ke BPM?

‚ùå **Tidak ada handling error/noise**
‚ùå **Tidak ada algoritma validasi data** (apakah sinyal valid atau noise?)

**YANG SEHARUSNYA ADA:**

**A. Algoritma Preprocessing (PENTING!):**
```
1. Baca raw PPG signal dari MAX30102
2. Apply DC removal (remove baseline drift)
3. Apply bandpass filter (0.5-4 Hz untuk heart rate)
   - Option 1: Moving Average Filter (window=10)
   - Option 2: Butterworth Filter (order=4)
   - Option 3: Kalman Filter
4. Normalize signal (0-1)
```

**B. Algoritma Peak Detection:**
```
1. Find local maxima in filtered signal
2. Apply minimum distance constraint (0.4s = 150 BPM max)
3. Apply amplitude threshold (>50% of max amplitude)
4. Count peaks in 10-second window
5. Calculate BPM = (peaks / window_duration) * 60
6. Apply smoothing (exponential moving average)
```

**C. Algoritma Validation:**
```
IF BPM < 40 OR BPM > 180:
    Mark as invalid
    Display "Error: Please retry"
ELSE:
    Accept BPM
    Display on OLED
    Send to IoT platform
```
